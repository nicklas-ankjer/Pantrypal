**original_problem_statement**: The user wants to build a mobile application called Kitchen Counter to manage household inventory, recipes, shopping lists, and emergency supplies. The application has evolved to include a multi-user household system for sharing all data, push notifications for expiring items, and advanced categorization for stock and shopping lists.

**PRODUCT REQUIREMENTS**:
-   **Core Functionality**: Track home stock, manage recipes, create shopping lists, and monitor emergency supplies.
-   **Household System**: Users can register with a username and PIN, create a Household, and invite other users via a generated code. All data (stock, recipes, lists) must be shared in real-time within the household.
-   **Recipes**: Users manage recipes. The app should show ingredient availability from Home Stock, considering items might exist in multiple locations. A Cook Recipe function deducts ingredients from stock.
-   **Shopping List**: Items can be added manually or from recipes. The list should be categorizable by user-defined store names and sorted/grouped by those stores. Users want to be able to re-categorize items easily.
-   **Home Stock**: Items have safety stock levels and can be organized into user-defined locations/categories.
-   **Emergency Stock**: Tracks items with expiration dates.
-   **Notifications**: The app should send push notifications to all household members when an emergency stock item is 30, 14, or 7 days from expiring. (This feature is currently blocked).

**User's preferred language**: English
